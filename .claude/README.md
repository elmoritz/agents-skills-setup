# .claude

Project-scoped Claude configuration.

- `agents/` — custom subagent definitions.
- `commands/` — custom slash commands.
  - `commands/ticket/` — ticket workflow entry points (`/ticket:init`, `/ticket:new`, `/ticket:refine`, `/ticket:pick`, `/ticket:review`, `/ticket:close`). They delegate all reads, writes, and transitions to the `ticket-engine` skill, which dispatches on `.claude/config.yaml`.
- `config.yaml` — project-scoped workflow configuration (generated by `/ticket:init`). Defines the lifecycle stages, the backend (filesystem or github), commit conventions, milestone strategy, types, effort caps, references, and verification commands.
- `references/` — static reference material loaded on demand.
- `skills/` — auto-triggered review/authoring skills plus the workflow engine.
  - `skills/ticket-engine/` — the project-agnostic execution layer the `/ticket:*` commands call into.
  - `skills/milestone-sync/` — milestone drift detection; dispatches on the configured strategy.
- `settings.json` — project-scoped settings overrides.

For project context, conventions, and roadmap, see `../CLAUDE.md` and `../docs/` (paths configurable via `config.yaml`'s `references:` block).
